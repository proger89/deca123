# Локальная выборка Thingi10K

После запуска `node tools/prepare_thingi10k_sample.cjs` здесь появляются 60 реальных STL-моделей: по 20 файлов в папках `B`, `C` и `D`.

- `B` — стандартный поток: все размеры меньше 450 мм и `K ≤ 0,80`.
- `C` — габаритный контроль: хотя бы один размер равен или больше 450 мм. Проверка размера выполняется раньше проверки формы.
- `D` — круглый товар: все размеры меньше 450 мм и `K > 0,80`.

Ожидаемый маршрут рассчитан тем же кодом `web/app.js`, который работает в браузерном демо. Полные измерения, лицензия исходного объекта, ссылки и SHA-256 записаны в `manifest.csv` и `manifest.json`.

Важно: формат STL не хранит единицы измерения. Для конкурсной проверки координаты трактуются как миллиметры. Поэтому разметка B/C/D верна именно при этом допущении.

Сами STL и исходные таблицы Thingi10K хранятся только локально и исключены из Git. У каждого объекта Thingi10K своя лицензия; перед публикацией конкретного файла нужно проверить поле `license` в манифесте.

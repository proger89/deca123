# Расчёты и связь с симуляцией

Документ отделяет официальные правила, аналитические формулы, численные physics-proxy и фактический Webots smoke. Число из proxy или discrete-event модели не называется физическим Webots-результатом.

## 1. Правило B/C/D

Пусть измеренные габариты отсортированы по убыванию: `d1 ≥ d2 ≥ d3`.

Габаритный допуск проходит только при одновременном выполнении:

```text
d1 > 10 мм и d2 > 10 мм и d3 > 10 мм
d1 < 450 мм, d2 < 320 мм, d3 < 320 мм
```

Любое равенство 10, 450 или 320 мм не проходит строгую границу и даёт C. Форма проверяется только после габаритов, поэтому большой круглый товар остаётся C.

Для замкнутого поперечного сечения:

```text
K = r_inscribed / R_circumscribed
```

где `r_inscribed` — минимальный радиус от центра сечения до контура, `R_circumscribed` — максимальный. После успешной проверки габаритов `K > 0,8` даёт D, а `K ≤ 0,8` — B.

Контрольные точки 9,99/10,00/10,01; 319,99/320,00/320,01; 449,99/450,00/450,01 и K=0,799/0,800/0,801 зафиксированы в `criteria/acceptance-contract.json`.

## 2. Консервативные интервалы

Перед испытанием зафиксированы половины интервалов:

- габариты: ±5 мм;
- K: ±0,03;
- после повторного сканирования: ±2 мм и ±0,015 K.

Если интервал пересекает габаритный порог, B запрещён и выбирается `ABSTAIN_DIMENSION → C`. Если пересекается 0,8 по форме, выбирается `ABSTAIN_SHAPE → D`. Это внутренние безопасные состояния; в официальной точности они считаются ошибкой, а не «правильным отказом».

Текущий `artifacts/phase-7/uncertainty-summary.json` — численный uncertainty harness на 1200 случаях, не Webots-замер. Нижний датчик в release отключён: улучшение p95 12,5% и снижение abstain на 0,6 п.п. не достигли заранее заданного порога полезности.

## 3. Возврат пассивной заслонки

Численная physics-proxy модель использует уравнение затухающего вращения без питания:

```text
I·θ'' + c·θ' + k·θ = 0
```

Параметры прототипа и модели:

| Параметр | Значение |
|---|---:|
| Инерция `I` | 0,12 кг·м² |
| Пружина `k` | 18 Н·м/рад |
| Демпфирование `c` | 2,35 Н·м·с/рад |
| Рабочий угол | 1,20 рад |
| Жёсткие упоры | 0…1,25 рад |
| Максимальный момент | 35 Н·м |
| Допуск датчика положения | 0,02 рад |

Собственная частота и коэффициент затухания:

```text
ωn = sqrt(k/I) = sqrt(18/0,12) ≈ 12,247 рад/с
ζ  = c / (2·sqrt(k·I)) ≈ 0,800
```

Для недодемпфированного возврата применена формула времени первого безопасного достижения положения покоя. Аналитическое значение в `artifacts/phase-9/mechanics.json` — 0,33944 с; численное интегрирование с шагом 0,002 с — максимум 0,342 с. Относительное отличие:

```text
|0,342 - 0,33944| / 0,33944 × 100% ≈ 0,753%
```

Это сравнение аналитики и численной physics-proxy, а не 200 Webots-прогонов. В Webots отдельно проверяется штатный physical smoke; полная Webots fault-квалификация должна иметь собственные трассы положения/момента.

## 4. Пропускная способность

Из скорости ленты 1 м/с и шага позиции 0,7 м следует:

```text
T = 0,7 / 1 = 0,7 с/товар
Q = 3600 / 0,7 = 5142,857 товара/ч
```

Актуальный Docker-артефакт `artifacts/p0-docker-throughput-final/throughput-report.json` содержит 5143 входа и 5143 выхода, нулевую конечную очередь, максимум 2 ожидающих товара и максимум 3 записи в ledger. Численно полученный цикл — 0,675259446 с против 0,700000000 с по формуле; отличие 3,534365%. После внесённого замедления очередь восстанавливается за 1,994452 с, p99 вычислительной модели FrameBundle — 107,429435 мс, а доля решений до дедлайна — 1,0.

Это аналитический плюс дискретно-событийный proxy-результат. Он не является часовым Webots-прогоном и не равен пропускной способности реального оборудования. `real_time_factor=17077,064` показывает лишь отношение модельного времени к wall-clock времени исполнения лёгкой discrete-event программы в CPU-контейнере с двумя доступными потоками; это число не переносится на Webots.

Для 7200 товаров/ч потребовался бы шаг:

```text
spacing = 3600 × 1 / 7200 = 0,5 м
```

Полная квалификация контактов, очереди и безопасности для 0,5 м отсутствует, поэтому `artifacts/p0-docker-throughput-final/claim-status.json` честно содержит `UNSUPPORTED`.

## 5. Ошибки геометрии

Финальный численный reliability harness использует пятиракурсный sensor model и continuous-time conveyor proxy. Пересчитанная сводка содержит:

- 10 564 proxy-маршрута;
- accuracy/coverage 0,9985806975 при 14 консервативных abstain из 9864 классифицируемых;
- mixed 3486/3500 = 0,996;
- p95 ошибки габаритов 0,578523 мм;
- p95 ошибки K 0,00813178;
- unsafe-to-B, lost и duplicates — 0 внутри этого harness.

Эти значения вычислены из синтетических модельных сенсорных данных и непрерывной модели движения. Они не являются результатом 10 564 Webots-прогонов и не доказывают фактическую обработку 11 выданных STL. Отдельное доказательство для реальных STL и RangeFinder возникает только из post-calibration item traces с первичными кадрами, решением и совпавшим датчиком выхода. Пока такие финальные трассы не прошли gate, статус этой части — «не заявлено».

## 6. Правило сравнения расчёта и модели

Для каждой величины используется единая формула:

```text
delta_percent = |simulation - analytical| / analytical × 100%
```

Порог цикла — ≤5%; actuation/stopping — <10%. В итоговом отчёте рядом с каждым числом обязательно указываются `evidence_type`, scenario ID, seed, исходный файл и хэш. Отсутствие любого из них блокирует публичное заявление.

# Инженерная проверка плана SafeSort

Проверка выполнена по инженерному контуру gstack после реализации плана и с учётом независимого разбора LLM Council.

## Архитектура

- Браузерный демонстратор и Webots не выдают себя за один и тот же контур. Браузер быстро разбирает пользовательский STL и объясняет решение, а Webots отдельно доказывает движение, работу заслонок и подтверждение выхода.
- Runtime отделён от evaluator: имя файла, эталонный класс и данные Supervisor не участвуют в решении.
- Разрешение B требует полного валидного измерения и двух подтверждённых положений заслонок. Неполные данные и отказ переводят товар в безопасный маршрут или останавливают линию.

## Качество кода и контракты

- Официальные правила B/C/D отделены от собственных инженерных решений SafeSort.
- Решение, физический маршрут и факт выхода — разные поля одного проверяемого события.
- Ошибки показаны оператору понятным текстом: «товар не определён», «маршрут не создан», «подтверждение выхода не получено».

## Проверки

```text
STL -> разбор -> измерение -> правило B/C/D -> маршрут -> заслонки -> датчик выхода
 |        |          |              |             |          |             |
 |        |          |              |             |          +-- отказ/таймаут
 |        |          |              |             +-- неверное положение
 |        |          |              +-- граница/неполные данные
 |        |          +-- невалидная геометрия
 |        +-- ASCII/binary, лимит размера и числа треугольников
 +-- неизвестное имя и источник
```

- Модульные, контрактные, физические и браузерные сценарии покрывают штатные B/C/D, неизвестный STL, неверный файл, повторную загрузку, частичный отчёт, отказ, подмену контрольной суммы и мобильный экран.
- Надёжность проверена на 10 564 маршрутах; опасных направлений в B, потерь и повторных действий не зафиксировано.
- Поток 5 143 товара/ч подтверждён с нулевым конечным хвостом. 7 200 товаров/ч честно отмечены как неподтверждённые.

## Производительность и эксплуатация

- Вход ограничен 50 МБ и 400 000 треугольников; тяжёлая геометрия обрабатывается ограниченно и не блокирует безопасное решение.
- Docker-запуск автоматически использует NVIDIA GPU, если он доступен, и сохраняет CPU-режим как обязательный запасной путь.
- Контейнер проходит healthcheck до открытия браузера. Адрес, порт и режим ускорения печатаются в консоль.

## Итог

Критических или нерешённых инженерных решений нет. Объём первого этапа зафиксирован: локальное демо, воспроизводимая Webots-симуляция, проверяемые метрики и комплект материалов для жюри. Загрузка на портал и зеркалирование в выданный организаторами репозиторий остаются действиями авторизованного участника.

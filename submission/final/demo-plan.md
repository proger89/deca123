# План демонстрации для жюри

## Главная режиссёрская идея

Центр демо — виртуальный конвейер Webots. Браузер используется первые 45–60 секунд только для ответа на самый естественный вопрос жюри: «Можно загрузить наш неизвестный STL?» После локального разбора файла защита сразу переходит к Webots-трассе, прошедшей post-calibration gate; исторический smoke после изменения датчиков показывать как доказательство нельзя.

```text
чужой STL в браузере → прозрачное правило B/C/D
→ неизвестный STL в Webots → 5 видов → решение
→ две физические заслонки → подтверждённый выход
→ отказ → безопасное состояние → проверяемые evidence-файлы
```

Нельзя говорить, что браузерный файл и Webots-товар совпадают, если их SHA-256 не совпадает в manifest. Корректная формулировка: «Браузер показывает локальный анализ произвольного STL; следующий Webots smoke независимо показывает sensor-to-exit цикл на неизвестном переименованном STL».

## Экран 1. Собственный STL жюри — 45–60 секунд

1. Показать пустое русскоязычное состояние без лишних панелей.
2. Попросить жюри выбрать STL или взять заранее переименованный файл с нейтральным именем.
3. Подчеркнуть: файл остаётся в браузере, имя не участвует в решении.
4. Повернуть 3D-модель, показать три габарита, K, применённую границу и итог B/C/D.
5. Запустить короткую схему движения и скачать JSON-результат.

Говорим: «Это нулевой порог входа и проверка неизвестного файла, а не доказательство Webots-физики».

## Экран 2. Виртуальный конвейер Webots — 80–100 секунд

Показать один непрерывный кадр или заранее проверенное непрерывное видео:

1. товар появляется в A;
2. видны пять RangeFinder и один общий tick/encoder position;
3. поверх кадра появляются измерения, interval и reason code;
4. первая и вторая заслонки показывают rest/permit и PositionSensor;
5. товар физически достигает выбранной зоны;
6. только после датчика выхода статус меняется на `SUCCESS`;
7. рядом остаётся `item_seq`, чтобы решение нельзя было приписать соседу.

Минимально валидный live proof — новый B smoke с актуальным calibration hash, read-only evaluator и совпавшим датчиком выхода. Для утверждения, что B/C/D физически показаны, нужны три реальные Webots-трассы с совпавшими exit sensor; объясняющая браузерная анимация это не заменяет.

## Экран 3. B, C и D — 25–35 секунд

Использовать трёхкадровый монтаж только из проверенных трасс:

- B: габариты проходят, `K ≤ 0,8`, обе заслонки разрешили проход;
- C: нарушена строгая граница, первая заслонка уходит в C;
- D: габариты проходят, `K > 0,8`, вторая заслонка уходит в D.

На каждом кадре одновременно видны expected rule, requested route и confirmed exit. Если C/D существуют только в unit/numerical harness, они маркируются как «logic/proxy», а не Webots physical.

## Экран 4. Отказ — 30–40 секунд

Лучший отказ для защиты — потеря питания/момента:

1. показать исчезновение drive;
2. показать возврат первой заслонки к C или второй к D;
3. показать, что B permit исчез;
4. следующий pocket остаётся в HOLD;
5. восстановление требует явного reset.

Если видеоряд построен по numerical physics-proxy, подпись должна быть такой же. Для слова «Webots fault» нужен trace из Webots с angle/torque/exit.

## Экран 5. Доказательства — 30–40 секунд

Открыть единый report и показать:

- тип доказательства рядом с каждой метрикой;
- commit, image digest, seed и `network=none`;
- source-event hash в JSONL/CSV/report/KPI;
- три semantic hashes;
- checksum tamper canary;
- статус 7200/ч `UNSUPPORTED` как пример честного claim gate.

## Пять Chrome-потоков до защиты

| Поток | Действия | Успех |
|---|---|---|
| Supplied STL | загрузить один выданный файл, повернуть, получить результат, скачать JSON | нет console/network error; решение объяснено |
| Renamed unseen STL | переименовать неизвестный файл, повторить | решение не зависит от имени |
| Boundary | загрузить/выбрать объект около границы | строгая граница/неопределённость видна, B не угадывается |
| Invalid | выбрать не-STL или повреждённый файл | русская ошибка, старый результат очищен |
| Retry | после invalid загрузить валидный STL | полный успешный цикл без перезагрузки страницы |

Дополнительно пройти только клавиатурой: focus видим, label связан с input, Enter/Space работают, canvas имеет текстовое описание, контраст проверен.

## Релизный gate видеодемонстрации

Основное видео принимается только если в нём визуально доказаны и не перепутаны:

- неизвестный/переименованный STL;
- пять синхронных видов;
- B/C/D с честной маркировкой типа доказательства;
- обе пассивно-безопасные заслонки;
- подтверждённый датчик выхода;
- хотя бы один отказ и безопасная реакция;
- измеряемые метрики с provenance;
- отсутствие склеек внутри заявленного непрерывного physical cycle.

Если любой пункт не имеет реального материала, его нельзя маскировать анимацией: лучше явно назвать текущую границу и показать воспроизводимую команду.
